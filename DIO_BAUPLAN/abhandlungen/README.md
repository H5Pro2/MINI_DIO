# Mini-DIO Abhandlungen

Dieser Ordner sammelt zusammenhängende Abhandlungen aus den Erkenntnissen des
Mini-DIO-Aufbaus.

Ziel ist nicht Laufdokumentation und nicht Codebeschreibung, sondern die
fachliche Verdichtung:

```text
Was wurde erkannt?
Welche Mechanik steckt dahinter?
Welche Grenze gilt?
Welche Bedeutung hat das fuer den weiteren DIO-Aufbau?
```

## Struktur

```text
abhandlungen/
  README.md
  01_MINI_DIO_ERKENNTNISSE.md
  02_PASSIVE_SEMANTIK_KANDIDATENKARTE.md
  03_DOKU_RELEVANZ_MINI_DIO_MCM.md
  04_KLEINER_DURCHBRUCH_MCM_TOPOLOGIE.md
```

## Aktive Abhandlungen

- `01_MINI_DIO_ERKENNTNISSE.md`  
  Erkenntnisse aus Mini-DIO bis zum aktuellen Stand:
  Wahrnehmung, MCM-Feld, passive Zeichenbildung, Fragmentinseln,
  unterbewusste Aktivitaetsinseln, neurologische und kosmologische Deutung
  sowie Uebertragbarkeit auf verkörperte Systeme.

- `02_PASSIVE_SEMANTIK_KANDIDATENKARTE.md`  
  Nächster Erkenntnis- und Ordnungsrahmen nach der Fragmentwiederkehr:
  Trennung von sensorischem Kern, affektiver Feldwirkung,
  akustisch-energetischem Anteil, Reflexionsanteilen, Randfragmenten und
  Rauschen. Bleibt ausdrücklich passiv und ohne Handlungswirkung.

- `03_DOKU_RELEVANZ_MINI_DIO_MCM.md`  
  Auswahl- und Ordnungspruefung fuer eine spaetere Mini-DIO-Dokumentation:
  relevante lokale Befunde, MCM-Theorieanker, Doku-Kern, Grenzen und
  vorgeschlagene Struktur fuer ein moegliches `MINI_DIO_PROJEKT`.

- `04_KLEINER_DURCHBRUCH_MCM_TOPOLOGIE.md`  
  Dokumentiert den kleinen Durchbruch als passiven MCM-Topologie-Befund:
  Zielraeume erscheinen als Zentrum, Bruecken, Driftspur und Spannungsrand.

## Abgrenzung

Die Abhandlungen sind keine Implementierungsanweisung im engen Sinn.
Sie sind die fachliche Theorieebene ueber dem Bauplan.

Der Bauplan liegt weiter hier:

```text
DIO_BAUPLAN/konstruktion/
```

Die Abhandlungen duerfen spaeter als eigener Bereich oder eigenes Repository
veroeffentlicht werden, zum Beispiel als:

```text
MINI_DIO_PROJEKT
```

Dabei muss die Grenze erhalten bleiben:

```text
Abhandlung = Erkenntnis / Theorie / Einordnung
Bauplan = Konstruktion / Schichten / Schnittstellen
Code = konkrete Umsetzung
```
