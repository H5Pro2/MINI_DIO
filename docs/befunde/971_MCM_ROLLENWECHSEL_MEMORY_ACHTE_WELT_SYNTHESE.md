# MCM-Synthese: Rollenwechsel-Memory gegen achte Welt

## Zweck

Diese Synthese verbindet die achte SOL-Multitimeframe-Weltgruppe 962 bis 970.
Geprueft wurde, ob die passive `dio_shift_*` Rollenwechsel-Memory aus 961 in einer weiteren Welt erneut sichtbar wird.

## Befund

Die achte Weltgruppe zeigt eine starke Fragmentierung:

- 5995 Verdichtungszonen,
- 5085 junge Spuren,
- 585 hoeherer Cluster-Uebergang,
- 183 randnahe Verdichtungen,
- 71 Rekopplungszonen,
- 41 Driftzonen,
- 30 stabile Bedeutungsinseln.

Die Brueckenlesung findet erneut keine Brueckenlandschaft:

- 0 Brueckenkanten,
- 0 Brueckenkerne,
- 0 Anschlussanker.

Die Nicht-Bruecken-Lesung zeigt dagegen eine breite Oberflaechenordnung:

- 2753 offene Nicht-Bruecken-Zeichen,
- 1672 Randspannungszeichen,
- 1445 schwache Zentrumszeichen,
- 91 mehrdeutige Oberflaechen,
- 26 Driftfelder,
- 5 Rekopplungsfelder,
- 3 offene Wahrnehmungsinseln.

Die `dio_shift_*` Rollenwechsel-Memory aus 961 wird in dieser Welt nicht direkt wiedergefunden:

```text
shift_spur_nicht_sichtbar = 8 von 8
```

## Interpretation

Dieser Befund sollte nicht als einfache Widerlegung der Rollenwechsel-Memory gelesen werden.

Die achte Welt ist strukturell anders als die siebte Welt:

```text
Siebte Welt:
  Verdichtung vorhanden,
  keine Bruecken,
  alte Reifungstokens bleiben als Nicht-Bruecken-Ordnung sichtbar.

Achte Welt:
  extreme Fragmentierung,
  sehr viele junge Spuren,
  keine Bruecken,
  alte Rollenwechsel-Tokens werden nicht direkt wiedergefunden.
```

Damit entsteht eine neue Differenz:

```text
Rollenwechsel-Welt != Fragmentierungswelt
```

In der siebten Welt wurden alte Zeichen umorganisiert.
In der achten Welt scheint die Feldoberflaeche so fein fragmentiert, dass alte Rollenwechselzeichen nicht als direkte Tokens wiederkehren.

## Bedeutung fuer MINI_DIO

MINI_DIO braucht hier eine weitere Unterscheidung:

```text
Rollenwechsel:
  altes Zeichen bleibt sichtbar,
  wechselt aber Organisationsform.

Fragmentierung:
  alte Zeichen werden nicht direkt sichtbar,
  die Welt bildet sehr viele junge, offene oder randnahe Spuren.
```

Das ist wichtig fuer die Memory:

- Nicht jeder Rollenverlust ist Feldverlust.
- Nicht jeder Feldverlust ist sofort Widerlegung.
- Eine stark fragmentierte Welt kann alte Zeichen verdecken, statt sie sauber weiterzufuehren.
- Junge Spuren koennen eine neue Bedeutungsoberflaeche bilden, bevor stabile Rollen entstehen.

## Arbeitsdeutung

Die achte Welt wirkt wie eine Oberflaechen- oder Streuungswelt.
Sie bildet viel Rohordnung, aber wenig wiedererkennbare Rollenbindung.

Das spricht dafuer, dass MINI_DIO mindestens drei Ordnungsarten unterscheiden sollte:

1. **Brueckenordnung**
   Bedeutungen verbinden sich ueber Anschlussrollen.

2. **Rollenwechselordnung**
   Bedeutungen bleiben sichtbar, wechseln aber ihre Feldrolle.

3. **Fragmentierungsordnung**
   Viele junge Spuren entstehen, alte Zeichen werden nicht direkt wiedergefunden.

## Schlussfolgerung

Die `dio_shift_*` Memory ist sinnvoll, aber sie ist nicht ausreichend fuer stark fragmentierte Welten.
Fuer solche Welten braucht es eine passive Fragmentierungs-Memory:

```text
Wie stark zerfaellt die Feldoberflaeche?
Welche jungen Spuren entstehen wiederholt?
Wann verdichten sie sich spaeter zu Rollen?
```

## Wie es weitergeht

Als naechstes sollte eine passive Fragmentierungs-Memory entstehen.
Sie speichert nicht einzelne alte Rollenwechsel, sondern Oberflaechenzustand:

```text
junge Spur,
offene Oberflaeche,
Randspannung,
schwaches Zentrum,
Rekopplungsrest,
Fragmentierungsgrad.
```

Damit kann spaeter geprueft werden, ob aus Fragmentierung neue Bedeutungsinseln wachsen.
