# MCM-Verdichtungspfade Klassifikation

## Zweck

Diese Synthese verbindet:

- `857_MCM_VERDICHTUNGSZONEN_DRIFTLUPE`
- `858_MCM_VERDICHTUNGSZONEN_TOKENLUPE`
- `859_MCM_TOKEN_ROHWELT_SEGMENTLUPE`
- `860_MCM_DRIFT_GEGEN_LOKALKONTAKT`
- `861_MCM_TOKEN_NACHBARSCHAFTSLUPE`
- `862_MCM_TOKEN_GRENZLUPE`

Ziel ist eine erste passive Klassifikation der vier untersuchten MCM-Episodentokens als Feldpfade.

## Grundbefund

Die Nachbarschaftslupe zeigte zunächst starke Selbstnachbarschaft. Das war methodisch erwartbar, weil lange zusammenhängende Tokenphasen Tick für Tick wieder sich selbst als Nachbarn sehen.

Die Grenzlupe ist deshalb fachlich wichtiger:

```text
Nicht jeder Tick zählt als Nachbarschaft.
Entscheidend ist: Wer führt in ein Segment hinein, wer führt heraus?
```

Dabei wurde sichtbar:

```text
Alle vier Tokens bilden stabile Innenphasen.
Aber der Austritt aus diesen Phasen wirkt im Mittel öffnend/belastend.
```

## Pfadklassen

### `dio_mcm_episode_0v5p8er`

Klassifikation:

```text
offener Driftpfad mit gegenseitiger Randnachbarschaft
```

Begründung:

- globale Zone wechselt von `hoeherer_cluster_uebergang` zu `driftzone`,
- reale Weltbreite nimmt ab,
- Folgebindung wird stärker synthetisch/randnah,
- dominanter Eintritt und Austritt koppeln an `14l8khu`,
- Austritt senkt Rekopplung und hebt Unruhe/Unschärfe.

Lesart:

`0v5p8er` ist kein isolierter Token. Er wirkt wie ein Teil eines driftenden Zweierfeldes mit `14l8khu`.

### `dio_mcm_episode_14l8khu`

Klassifikation:

```text
offener Driftpfad mit lokaler Rekopplungsmaske
```

Begründung:

- globale Zone wechselt von `hoeherer_cluster_uebergang` zu `driftzone`,
- lokale Segmentlupe wirkte zunächst rekoppelnd,
- Grenzlupe zeigt aber: Eintritt und Austritt koppeln dominant an `0v5p8er`,
- Austritt senkt Rekopplung deutlich und hebt Strain/Lautheit.

Lesart:

`14l8khu` zeigt den wichtigen Unterschied zwischen lokalem Kontakt und globalem Pfad. Lokal kann der Kontakt beruhigt wirken; als Feldpfad bleibt er driftend, weil seine Nachbarschaft öffnet.

### `dio_mcm_episode_1q3us3f`

Klassifikation:

```text
rekoppelnder Innenpfad mit öffnendem Austritt
```

Begründung:

- globale Zone verschiebt sich von stabiler Bedeutungsinsel zu `rekopplungszone`,
- lokale Segmentlupe bestätigt Rekopplung klar,
- Weltbreite bleibt erhalten,
- dominanter Eintritt/Austritt koppelt an `18l3thm`,
- Austritt wird trotzdem öffnend/belastend.

Lesart:

`1q3us3f` ist kein reiner Stabilitätspunkt. Er trägt Rekopplung im Inneren, aber der Übergang aus ihm heraus führt in eine offenere Feldlage.

### `dio_mcm_episode_1xx3u1e`

Klassifikation:

```text
stabile Bedeutungsinsel mit belastetem Randübergang
```

Begründung:

- globale Zone verschiebt sich von `hoeherer_cluster_uebergang` zu `stabile_bedeutungsinsel`,
- Weltbreite steigt,
- Zentrum bleibt dominant,
- dominanter Eintritt/Austritt koppelt an `0ybr5e3`,
- Austritt bleibt leicht öffnend/belastend.

Lesart:

`1xx3u1e` wirkt wie eine stabile Insel, deren Randübergang noch Last trägt. Die Bedeutung stabilisiert sich nicht durch einen einzelnen starken Kontakt, sondern durch Weltbreite und Rollenbindung.

## Gesamtbild

Die vier Tokens bilden keine einfache Liste von Zeichen. Sie wirken wie unterschiedliche Feldpfade:

```text
0v5p8er  -> driftender Gegenpfad
14l8khu  -> driftender Gegenpfad mit lokaler Rekopplungsmaske
1q3us3f  -> rekoppelnder Innenpfad
1xx3u1e  -> stabile Bedeutungsinsel
```

Damit wird die aktuelle MCM-Arbeitsthese konkreter:

```text
MCM-Bedeutung entsteht nicht nur durch Wiederkehr.
Sie entsteht durch wiederkehrende Feldpfade:
Kontakt, Innenphase, Austritt, Nachbarschaft und Weltbreite.
```

## Bedeutung Für MINI_DIO

MINI_DIO beginnt dadurch, Bedeutungen nicht nur als Token zu tragen, sondern als Bewegungsform:

- ein Zeichen kann eine Insel sein,
- ein Zeichen kann ein Pfad sein,
- ein Zeichen kann eine Brücke sein,
- ein Zeichen kann eine Driftzone sein,
- ein Zeichen kann lokal tragen und global trotzdem öffnen.

Das ist ein wichtiger Unterschied zur reinen Symbolstatistik.

## Wie es weitergeht

Als nächstes sollte diese Pfadklassifikation auf mehr Tokens erweitert werden. Ziel: prüfen, ob sich stabile Kategorien bilden:

```text
stabile Insel
rekoppelnder Pfad
offener Driftpfad
Randpfad
Brückenpfad
```

Wenn diese Kategorien über weitere Tokens und Welten reproduzierbar bleiben, wäre das ein starker Hinweis auf eine eigenständige MCM-Feldpfadstruktur.
